# RippleFlow 技术文档索引

## 系统概述

**RippleFlow** 是一款面向创业软件开发团队的自用效率工具，核心定位为：
> 让团队的群聊历史变成一个会思考、会回答、会自动整理的活知识库

### 核心价值
- **信息平权**：无论何时加入团队，均可获取历史决策与讨论上下文
- **知识沉淀**：对话自动结构化为 FAQ、决策日志、故障案例、参考手册
- **反复对齐消除**：LLM 驱动的智能问答，避免相同问题重复讨论
- **可信修正**：当事人可修正 LLM 总结，修正可选同步至原群
- **即时触达**：在群聊中直接@机器人查询，无需切换应用
- **主动运营**：AI 管家主动推送、提醒、发现价值，让知识库真正"活"起来

### 设计约束
| 约束 | 说明 |
|------|------|
| 聊天工具 | 自研，通过 Webhook 推送消息，机器人可回复但不主动发群 |
| 机器人客户端 | 聊天机器人作为 RippleFlow 第二客户端，支持群内自然语言查询 |
| AI 管家 | 独立的智能运营角色，可主动发群推送提醒和快报 |
| 认证 | 企业 LDAP + SSO，系统只做白名单管理 |
| LLM | 公司内部部署模型，全中文场景优化 |
| 存储 | PostgreSQL（无向量扩展），全文检索 + LLM 直接处理 |
| 敏感信息 | 发布前需全部当事人明确授权，pending 状态 7 天后升级管理员 |
| 检索策略 | 全文检索召回候选 + LLM 基于摘要综合回答（非语义向量检索） |
| 时间窗口 | 默认 3 个月滑窗（技术决策、参考信息永久有效） |

## 客户端入口

RippleFlow 提供三个客户端入口：

### 1. Web Dashboard（完整功能）

面向深度使用场景，提供完整的知识库管理能力：

| 功能模块 | 说明 |
|----------|------|
| 知识库浏览 | 按类别、时间、标签筛选话题线索 |
| 搜索问答 | 自然语言提问，返回答案 + 来源 |
| 敏感授权 | 处理待授权的敏感内容请求，实时查看授权进度 |
| 当事人修改 | 修正摘要内容，可选同步回群 |
| 个人贡献 | 查看个人知识贡献统计 |
| 管理后台 | 白名单、类别管理、统计面板（管理员） |

### 2. 聊天机器人（即时查询）

面向轻量查询场景，在群聊中直接@机器人：

```
用户：@机器人 Redis 连接池怎么配置
机器人：找到 2 条相关记录...

用户：@机器人 我有什么待办
机器人：您有 3 个待办任务...

用户：@机器人 生成今天产品群的会议纪要
机器人：已生成纪要，点击查看详情...
```

**机器人支持的查询类型**：
- 知识库搜索/问答
- 个人待办查询
- 参考数据查询（IP、URL、配置）
- 会议纪要生成（可指定时间范围）

### 3. AI 管家（平台灵魂）

AI 管家是 RippleFlow 的"灵魂"，通过**感知-决策-执行-自省**循环持续运营：

```
感知层：监控所有平台事件（消息、问答、待办、订阅、用户行为...）
    ↓
决策层：判断事件相关性、行动必要性、执行方式
    ↓
执行层：发送通知 | 生成报告 | 创建任务 | 回复问答
    ↓
自省层：分析效果 | 提取模式 | 沉淀最佳实践 | 优化提示词
```

**AI 管家核心职责**：

| 职责 | 说明 |
|------|------|
| 信息平权 | 确保信息公平触达每个需要的人 |
| 智能推荐 | 发现有价值的信息并推荐给相关人员 |
| 总结提炼 | 将复杂信息转化为易理解的形式 |
| 问答辅助 | 帮助用户快速找到答案 |
| 任务跟踪 | 跟踪任务进度，确保不遗漏 |
| 及时提醒 | 在关键时刻提醒相关人员 |

**AI 管家核心技能**：
- 数据可视化：自动识别群聊数据，生成图表
- 链接摘要：自动获取链接内容并生成摘要
- 任务识别：从群聊消息中识别任务并创建待办
- 偏好学习：记住用户偏好，自动应用到后续服务

## AI 管家架构（v2.0）

### 提示词分级架构

```
butler_prompts/
├── core/          # 冷启动核心提示词（不可修改）
├── duties/        # 职责定义（管家可优化）
├── skills/        # 技能模板（管家可优化）
├── insights/      # 自省沉淀（管家维护）
└── extensions/    # 扩展脚本（零代码新功能）
```

### 自省与平台迭代

| 周期 | 内容 |
|------|------|
| 每日自省 | 回顾行为效果，提取模式 |
| 每周复盘 | 汇总数据，优化职责 |
| 月度评估 | 平台整体评估，改进建议 |

AI 管家维护自己的经验知识库，持续优化服务质量：

```
butler_experience_memory:
  - 高峰使用时段: "周一上午问答量大"
  - 常见问题模式: "Redis配置相关占比30%"
  - 快报最佳发送时间: "周一9:30打开率最高"
  - 待办完成率: "提醒后完成率提升40%"
  - 问答满意度: "本月平均4.2/5分"
```
