## 四、架构设计亮点

### 4.1 策略与机制分离

```
知识平台（机制）         AI管家Agent（策略）
      ↓                        ↓
   稳定不变                  持续演化
   提供能力                  决定使用
   版本发布                  在线优化
```

### 4.2 Agent + Routine + 工具集

```
┌─────────────────────────────────────────────┐
│              AI 管家 Agent                   │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│  │ 感知    │→ │ 决策    │→ │ 执行    │     │
│  └─────────┘  └─────────┘  └─────────┘     │
│       ↑            ↑            ↓           │
│       └────────────┴────────────┘           │
│                 自省反馈                     │
├─────────────────────────────────────────────┤
│  Routine 脚本（固化逻辑的提示词形式）        │
│  • 敏感授权升级                              │
│  • 待办提醒                                  │
│  • 话题通知                                  │
│  • 每日快报                                  │
├─────────────────────────────────────────────┤
│  工具集（平台能力抽象）                      │
│  • search_threads                           │
│  • create_todo                              │
│  • send_notification                        │
│  • query_knowledge_graph                    │
│  • ...                                      │
└─────────────────────────────────────────────┘
```

### 4.3 信息态势感知

```
用户核心问题              系统能力
────────────              ────────
发生了什么？       ←→    全局索引、热点分析
和谁相关？         ←→    知识图谱、当事人识别
状态如何？         ←→    状态追踪、闭环检测
瓶颈在哪？         ←→    瓶颈识别、阻塞分析
如何解决？         ←→    决策记录、待办跟踪
需要什么资源？     ←→    资源提取、依赖分析
谁支持谁？         ←→    支持关系图谱
```

## 五、总结

本次 v0.6 架构重构完成了以下重大更新：

1. **架构分离**：知识平台与 AI管家Agent 完全解耦
2. **Agent 升级**：AI管家成为独立智能体，具备感知-决策-执行-自省能力
3. **Routine 机制**：固化逻辑改为提示词脚本，支持在线更新和演化
4. **工具调用**：平台能力抽象为工具，供 Agent 通过 Function Call 调用
5. **态势感知**：新增全局索引、信息链视图、瓶颈识别等核心能力
6. **知识图谱**：采用显式建模+Schemaless扩展的双层架构
